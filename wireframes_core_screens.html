Wireframe 1: Onboarding Flow

🏠
Welcome to People Empowering App
We're here to help you find housing in NYC. No confusing forms. No overwhelming lists. Just clear steps and real support.

Design Decisions - Onboarding Screen 1

Simple, friendly language: "No confusing forms. No overwhelming lists."
Research: Mr. L said "Don't make me more confused" - this sets expectations immediately.
Skip option: Respects user autonomy (trauma-informed design)
Visual metaphor: House emoji = housing (universal, accessible)
Progress dots: Shows 3 screens only - keeps it short
Research: 25% users are "somewhat uncomfortable" with apps - onboarding builds confidence
PEA

Design Decisions - Needs Assessment

Help button always visible: Header position, bright yellow
Research: 100% want option to speak with real person - accessible from first screen
Large touch targets: Easy to tap, reduces errors
Icons + text: Supports multiple literacy levels
Emergency prioritized: First option for users in crisis (Mr. L's situation)
"I'm not sure" option: No wrong answer, connects to human immediately
Research: 67% felt lost - this prevents that feeling from the start
Proactive suggestions start here: Based on selection, app will suggest resources

Design Decisions - Onboarding Complete

Immediate value: "We've created your personal action plan" - not just registration
Reassurance: "you can always chat with someone if you get stuck"
Research: Mr. L felt "lost and unsupported" - this promises he won't be
Clear CTA: "View My Action Plan" = specific, not vague "Continue"
Checkmark icon: Visual progress, builds confidence

Design Decisions - Action Plan View

Visual progress: Progress bar + "2 of 5" = multiple ways to understand
Research: 100% found step-by-step plans valuable - this is the core feature
Completed steps shown: Builds confidence, shows progress
Current step highlighted: Thick border, white background = clear focus
Plain language: "You need these 4 documents" not "Required documentation"
Specific, actionable: Lists exact documents, not vague categories
Research: 33% had missing documents as challenge - "I'm missing a document" button addresses this
Community tips integrated: Real advice from peers
Verification badge: 75% trust tips if verified - shows official confirmation
Multiple CTAs: Upload docs OR get help if missing - no dead end
"I'm stuck" button prominent: 100% want human support option

Design Decisions - Human Support Hub

Multiple support channels: Chat, phone, video, text
Research: Information preferences varied - 50% email, 25% each for other channels. Support must be multimodal.
Response time visible: Sets expectations, reduces anxiety
Availability status: "Available now" vs "24/7" - clear communication
FAQ before live support: Faster resolution for common questions
Emergency option prominent: Crisis users (like Mr. L) get immediate help
Research: 100% want option to speak with real person - this screen delivers on that promise
Button state change: "Get Help" → "Help Active" shows user is supported

Design Decisions - Live Chat

Context-aware greeting: "I see you're on Step 2..." - counselor knows user's situation
Named counselor: "Maria" not "Agent 437" - humanizes interaction
Role visible: "(Housing Counselor)" builds credibility
Empathetic response: "Yes! You have options." - positive, supportive
Clear alternatives: Offers two paths, asks user preference
Research: Mr. L said "Let me chat directly with someone" - this delivers exactly that
Voice-to-text option: Accessibility feature for users uncomfortable typing
Research: 50% use voice-to-text messaging - should offer in chat too
Connected status: Header shows "Connected" - reassures user
