Design System Overview

This design system provides a comprehensive guide for building the People Empowering App with consistency, accessibility, and user-centered design at its core.

Design Principles

🎯 Simplicity First

Every element serves a purpose. No confusion, no overwhelm. Based on Mr. L's request: "Don't make me more confused."

🤝 Human-Centered

Technology supports human connection, never replaces it. 100% of users want the option to speak with a real person.

♿ Accessible by Default

Design for all abilities from the start. 100% of participants rated accessibility as at least somewhat important, 50% as essential.

💪 Empowering

Give users control and clarity. Show progress, explain options, never create dead ends.

🔒 Privacy Protected

System forces survival strategies - we must protect user data absolutely and transparently.

📱 Mobile-First

Designed for smartphones. All participants have smartphones; 50% use advanced phone features.

Color Philosophy

Colors are chosen for:

  • Trust: Blue primary color conveys stability and reliability
  • Energy: Purple secondary adds warmth and community feel
  • Accessibility: All color combinations meet WCAG AA standards minimum
  • Semantic clarity: Green = success, Yellow = warning, Red = danger/crisis

Typography Philosophy

System fonts ensure:

  • Readability: Large base size (16px) for all users
  • Performance: System fonts load instantly, no web font delays
  • Familiarity: Users already know these fonts from their devices
  • Accessibility: Clear hierarchy with sufficient size differences

Typography Guidelines

  • Base size minimum 16px: Never go below base size for body text (accessibility)
  • Line height 1.6: Provides comfortable reading with sufficient spacing
  • Paragraph spacing: Use margin-bottom equal to 1 line height (1.6rem)
  • Hierarchy: Maintain clear size jumps between heading levels (minimum 4px)
  • Plain language: Use simple words, short sentences, active voice
  • No jargon: Replace technical terms with everyday language

Accessibility Standards

100% of participants rated accessibility as at least somewhat important, with 50% rating it as essential. Accessibility is not optional—it's fundamental.

WCAG Compliance

Target: WCAG 2.1 AA

All components must meet AA standards minimum. AAA preferred where possible.

Color Contrast

Minimum 4.5:1 for normal text, 3:1 for large text and UI components.

Touch Targets

Minimum 44x44px for all interactive elements (buttons, links, inputs).

Focus Indicators

Visible focus states for keyboard navigation. Never remove outlines without replacement.

Multimodal Accessibility

Based on research: 50% use photo documents, voice-to-text, translation apps.

  • Voice Input: Support voice commands for all text inputs
  • Photo Upload: Camera-first approach for document management
  • Screen Reader: All images have alt text, all interactive elements labeled
  • Translation Ready: Semantic HTML enables easy translation
  • Text Scaling: All layouts work from 100% to 200% text zoom
  • Color Independent: Never rely solely on color to convey meaning

Keyboard Navigation

Key Action
Tab Move focus to next interactive element
Shift + Tab Move focus to previous interactive element
Enter / Space Activate button or link
Escape Close modal or cancel action

Semantic HTML

  • Use <button> for actions, <a> for navigation
  • Use <nav>, <main>, <section> landmarks
  • Use heading hierarchy (h1 → h2 → h3) without skipping levels
  • Use <label> for all form inputs
  • Use ARIA attributes when semantic HTML isn't sufficient

Plain Language Requirements

Research finding: "Don't make me more confused" - Mr. L's top request

  • Reading Level: Target 8th grade reading level maximum
  • Sentence Length: Average 15-20 words, maximum 25 words
  • Active Voice: "Upload your documents" not "Documents should be uploaded"
  • No Jargon: "Proof of where you live" not "Residency verification"
  • Explain Acronyms: "DHS (Department of Homeless Services)" on first use
  • Positive Language: "You need 4 documents" not "You're missing 4 documents"

Design Checklist

  • All text meets 4.5:1 contrast ratio minimum
  • All interactive elements have 44x44px minimum touch target
  • All images have descriptive alt text
  • All forms have associated labels
  • Color is not the only way information is conveyed
  • Focus indicators are visible for keyboard navigation
  • Plain language is used throughout (8th grade level)
  • Human support option is always visible and accessible
  • Progress is clearly shown for multi-step processes
  • Error messages are clear, specific, and actionable
